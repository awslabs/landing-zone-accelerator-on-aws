    Type (Required)

    Determines whether an account-wide subscription is applied, or if a Lambda function will be invoked to apply each log group.

    Example

    type: ACCOUNT

    Optional ReadonlyselectionCriteria

    selectionCriteria?: string

    Selection Criteria (Optional)

    Selection criteria for the account-wide subscription. Only used when ICloudWatchSubscriptionConfig.type is 'ACCOUNT'. This should be used to Prevent log recursion.

    In the following example, log groups with the names /aws/lambda/AWSAccelerator-FirehoseRecordsProcessor, development, or AppA will not have a subscription filter.

    Example

    selectionCriteria: 'LogGroupName NOT IN ["/aws/lambda/AWSAccelerator-FirehoseRecordsProcessor", "development", "AppA"]'

    See

    PutAccountPolicy for more information on the expected format of selectionCriteria

    Optional ReadonlyoverrideExisting

    overrideExisting?: boolean

    Override Existing (Optional)

    Indicates whether the existing CloudWatch Log subscription configuration can be overwritten. If enabled, any existing policy will be updated and renamed to 'ACCELERATOR_ACCOUNT_SUBSCRIPTION_POLICY'. Upon deleting the solution or disabling logging for cloudwatch in global config, this policy will be removed. If type is set to 'LOG_GROUP' this parameter will not be used.

    Default

    false

    Optional ReadonlyfilterPattern

    filterPattern?: string

    Filter Pattern (Optional)

    The specific filter pattern to apply to the subscription. If no value is provided all logs events will match filter criteria. Only applicable when ICloudWatchSubscriptionConfig.type is 'LOG_GROUP'
